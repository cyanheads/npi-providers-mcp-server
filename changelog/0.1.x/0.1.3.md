# 0.1.3 — 2026-06-20

Framework-maintenance release: adopts `@cyanheads/mcp-ts-core` `^0.10.9` and re-syncs the bundled devcheck scripts and agent skills. No tool, resource, or behavior changes.

## Added

- **`Dependency Specifiers` devcheck step** — new `scripts/check-dependency-specifiers.ts`, hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **Plugin-manifest checks in `lint:packaging`** — validates `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json`/`mcp.json` for non-empty descriptions, unscoped machine name in display fields, and the full package name in the `npx -y` install arg; gated by the new `devcheck.config.json` `packaging.pluginManifests` flag. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))

## Changed

- **Bundled framework scripts re-synced** — `scripts/devcheck.ts`, `build-changelog.ts`, `check-framework-antipatterns.ts`, `check-skill-versions.ts`, and `lint-packaging.ts` adopt the 0.10.7 fresh-scaffold guards (git-dependent checks skip without `.git`; changelog sync exits clean on a template-only tree; skill-version check tolerates worktree-deleted skills) and the 0.10.9 packaging additions. ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237), [#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243))
- **Bundled skills re-synced** to the framework versions — `api-context` documents `ctx.content` (the image/audio collector); `api-config`/`api-telemetry` correct the `OTEL_SERVICE_NAME` seeding precedence; `git-wrapup`/`orchestrations`/`polish-docs-meta` and the seven equal-version skills carry their 0.10.7–0.10.9 body updates. ([cyanheads/mcp-ts-core#233](https://github.com/cyanheads/mcp-ts-core/issues/233), [#238](https://github.com/cyanheads/mcp-ts-core/issues/238), [#239](https://github.com/cyanheads/mcp-ts-core/issues/239))

# 0.1.0 — 2026-06-13

## Added

- **`npi_search_providers`** — search the NPPES NPI registry by name, organization, location, provider type, and specialty. Plain-language specialties resolve through the bundled NUCC taxonomy to the registry's exact descriptions before searching, and the resolved taxonomy is echoed back. Discloses the page-size-not-total caveat and the 1200-match pagination ceiling.
- **`npi_get_provider`** — decode up to 10 NPIs to full records (taxonomies, addresses, credentials, identifiers, endpoints, status), fanning out one call per NPI with partial-success reporting (`found` / `notFound`).
- **`npi_lookup_taxonomy`** — offline NUCC Healthcare Provider Taxonomy resolver: `resolve` (plain term → codes, with specialty-suffix stemming), `get` (exact code), `browse` (hierarchy, filterable by grouping and NPI section).
- **`npi://provider/{npi}`** and **`npi://taxonomy/{code}`** resources — read-only twins of the get/lookup tools.
- **`nppes-service`** — NPPES API v2.1 client. Detects the registry's HTTP 200 + `Errors[]` validation-failure envelope and maps it to typed contract reasons (`no_search_criteria`, `invalid_npi_format`, `invalid_search_field`); `withRetry` + `fetchWithTimeout` cover the full fetch/parse pipeline. Normalization preserves upstream sparsity.
- **`taxonomy-service`** — 879 NUCC codes (v25.0) bundled as a generated in-memory index, loaded at startup. Refresh via `scripts/generate-taxonomy-data.ts`.

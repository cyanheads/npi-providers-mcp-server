# 0.1.6 — 2026-07-11

## Added

- **`npi_lookup_taxonomy` `skip` input** — pages past a truncated `resolve`/`browse` result (0–1000, default 0; ignored for `get`). Keep the same query/filters and `limit`, raise `skip` by `limit` each call; the `truncated` guidance and the empty-page notice both point at it directly. ([#7](https://github.com/cyanheads/npi-providers-mcp-server/issues/7))

## Changed

- **Bundled NUCC taxonomy attribution** — added a root `NOTICE` file crediting the American Medical Association on behalf of the National Uniform Claim Committee (NUCC), a matching copyright banner in `taxonomy-data.ts` and its generator (`scripts/generate-taxonomy-data.ts`) so regeneration carries the notice forward, and a strengthened README attribution line linking the NUCC permission. ([#2](https://github.com/cyanheads/npi-providers-mcp-server/issues/2))

## Fixed

- **`npi_lookup_taxonomy` `resolve` plain-language specialties** — a query-side stop-word set (`doctor`, `physician`, `specialist`, `provider`, `md`, `do`) strips generic noise words so phrases like "heart doctor" and "eye doctor" reduce to a token the NUCC vocabulary actually carries, and an extended lay-term alias table (`heart`, `eye`, `ent`, `kidney`, `cancer`, `obgyn`) maps abbreviations and colloquialisms to their formal specialty. Aliased tokens now match as whole words, so bare "ent" reaches Otolaryngology instead of substring-hitting "gastroENTerology". ([#1](https://github.com/cyanheads/npi-providers-mcp-server/issues/1))

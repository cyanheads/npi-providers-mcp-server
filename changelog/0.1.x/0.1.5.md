# 0.1.5 — 2026-07-11

## Added

- **`npi_get_provider` `errored` output** — NPIs whose lookup failed with an upstream error (service unavailable, timeout) land in a new `errored` array, kept distinct from `notFound`'s confirmed absence. ([#8](https://github.com/cyanheads/npi-providers-mcp-server/issues/8))
- **`npi_get_provider` timestamps** — decodes `createdEpoch`/`lastUpdatedEpoch` (epoch milliseconds) on the full record. ([#9](https://github.com/cyanheads/npi-providers-mcp-server/issues/9))
- **`npi_get_provider` name and endpoint fields** — `otherNames[]` gains `middleName`/`prefix`/`suffix`; `authorizedOfficial` gains `namePrefix`/`nameSuffix`; `endpoints[]` gains `useDescription`, `contentTypeDescription`, `affiliation`, `affiliationName`, and a routing address block (`addressType`, `line1`, `city`, `state`, `postalCode`, `countryCode`, `countryName`). ([#9](https://github.com/cyanheads/npi-providers-mcp-server/issues/9))

## Fixed

- **`npi_get_provider` placeholder rendering** — the NPPES `"--"` sentinel on name prefix/suffix (individual, authorized official, other names) is now normalized to absence instead of rendered as a real value, via a shared `trimmedNonPlaceholder` guard. ([#6](https://github.com/cyanheads/npi-providers-mcp-server/issues/6))
- **`npi_get_provider` upstream-failure handling** — an upstream failure (service unavailable, timeout) no longer reports as a confirmed missing NPI; `getByNumber` results now partition into `found` / `notFound` (confirmed `result_count` 0) / `errored` (upstream failure, already retried), and an all-failed batch re-throws the real service error instead of the misleading `none_found`. ([#8](https://github.com/cyanheads/npi-providers-mcp-server/issues/8))
